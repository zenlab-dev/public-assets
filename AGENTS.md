# Agent guide — zenlab-dev/public-assets

This public repository is generated from
`zenlab-dev/zenlab-infra:scripts/public-assets`.

## Repo model

- `README.md` is copied from `scripts/public-assets/README.md`.
- `bootstrap/README.md` is copied from
  `scripts/public-assets/bootstrap/README.md`.
- `bootstrap/zenlab-host.sh` is copied from
  `scripts/public-assets/bootstrap/zenlab-host.sh`.
- `bootstrap/zenlab-host.sh.sha256` is generated by the publish workflow.

## Rules of the road

- Do not edit this public repo directly for managed files. Change the source in
  `zenlab-infra`, merge that PR, and let `CD - Publish public assets` sync it.
- Everything here must be safe for anonymous public read.
- Public SSH keys and bootstrap logic are OK. Private SSH keys, API tokens,
  encrypted secrets keys, decrypted `secrets.yaml`, generated `.env.ci`, and
  machine-specific passwords are not OK.
- Keep bootstrap scripts POSIX/Bash-compatible for fresh Linux hosts and avoid
  assumptions beyond root access, OpenSSH, Docker, and Tailscale being prepared
  as documented.

## Validation

From the `zenlab-infra` source repo, validate changes with:

```bash
bash -n scripts/public-assets/bootstrap/zenlab-host.sh
git diff --check
```

After publish, verify the anonymous raw URL and checksum:

```bash
tmp="$(mktemp -d)"
cd "$tmp"
curl -fsSLO https://raw.githubusercontent.com/zenlab-dev/public-assets/main/bootstrap/zenlab-host.sh
curl -fsSLO https://raw.githubusercontent.com/zenlab-dev/public-assets/main/bootstrap/zenlab-host.sh.sha256
sha256sum -c zenlab-host.sh.sha256
```
